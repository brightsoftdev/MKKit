MKTableCellDelegate Protocol Reference

Conforms to NSObject
Declared in MKTableCellDelegate.h

Overview

The MKTableCellDelegate is designed to work with an NSDictonary object. It uses the key property of MKTableCell to identify input changes and notify the delegate of them. This allows you to quickly process large sets of data input from the user.

Here is an example of how it works: 

- (void)valueDidChange:(id)value forKey:(NSString *)aKey {
	[myDictionary setObject:value forKey:aKey];
}

Tasks

Input Changes

  • - valueDidChange:forKey: Called when a the Value of a controled object is changed.

Responding to Events

  • - didSelectCell:forKey: Called when the cell is selected.
  • - didTapAccessoryForKey: Called when the cells Accessory View is touched.

    Note * This method is called when using one of the MKTableCellAccessoryType constances. If your using one of the UITableCellView accessory use those methods to process touches.

Discussion

Called when a the Value of a controled object is changed.

All of the sublcasses that use any type of user input contols call this method whenever the input value is changed. It is imporant that the key property of MKTableCell is set for this method to work propery.

Note * If a contol gives an integer or bool for its value, the value is converted into an NSNumber object before this method is called.
